# Docker 基础知识

标签：#技术知识 #Docker #部署 #容器化

## 一句话总览

Docker 用来把“程序 + 它需要的运行环境”一起打包，然后在不同机器上稳定运行。

它主要解决的是：

> 为什么程序在我电脑上能运行，换到别人电脑或服务器上就运行不了？

## Docker 是干嘛用的

一个 Python 项目可能依赖：

- 特定版本的 Python，例如 Python 3.11。
- 一组第三方库。
- MySQL、Redis 等外部服务。
- 特定的系统配置和环境变量。

如果直接运行，开发机、测试机和生产服务器的环境可能不同，最终出现“在本地能跑、部署后报错”的问题。

Docker 可以把这些运行条件封装起来，形成一个相对独立的运行单元。只要目标机器安装了 Docker，就可以用同一套镜像启动相同的应用环境。

## 可以怎么理解

Docker 类似物流中的集装箱：

- Docker 镜像：集装箱模板，包含程序、依赖和启动配置。
- Docker 容器：根据镜像实际启动的运行实例。
- Dockerfile：制作镜像的说明书。
- Docker Compose：描述并管理多个容器的配置文件。

镜像本身是静态的，容器是镜像启动后的动态实例。同一个镜像可以启动多个容器。

## 典型 Web 项目结构

一个网站可能拆成多个容器：

```text
前端容器
后端容器
MySQL 容器
Redis 容器
```

它们彼此隔离，但可以通过 Docker 网络互相访问。一个常见的请求链路是：

```text
用户浏览器
  -> Caddy 或 Nginx
  -> 后端应用容器
  -> MySQL 或 SQLite
```

- Caddy 或 Nginx 负责域名、HTTPS、反向代理和入口流量。
- 后端容器负责业务逻辑。
- MySQL 或 SQLite 负责保存业务数据。
- Redis 容器通常负责缓存、队列或临时状态。

Docker 负责把这些服务分别运行起来，并提供网络、端口、存储等基础隔离能力。

## Docker 和虚拟机的区别

虚拟机通常需要运行一个完整的操作系统，资源占用较大，启动也更慢。

Docker 容器通常共享宿主机的底层内核，只隔离应用进程及其依赖，因此启动更快、占用更少。

需要注意：Docker 本身不是虚拟机。macOS 和 Windows 上的 Docker Desktop 为了运行 Linux 容器，底层通常会启动一个轻量 Linux 虚拟机，但使用 Docker 时主要操作的仍然是容器。

## Docker 常见用途

- 本地开发：统一团队成员的运行环境。
- 测试：快速启动干净、可重复的测试环境。
- 部署：把开发环境中的镜像部署到服务器。
- 微服务：让不同服务分别运行、分别扩缩容。
- CI/CD：在自动化流水线中构建和测试应用。
- 依赖管理：避免在宿主机上安装大量版本相互冲突的软件。

## 常见概念之间的关系

```text
Dockerfile
  -> 构建
Docker 镜像
  -> 启动
Docker 容器
  -> 运行
应用服务
```

如果一个项目需要多个服务，可以用 Docker Compose 描述它们之间的关系：

```text
compose.yaml
  -> 后端容器
  -> MySQL 容器
  -> Redis 容器
```

Compose 适合本地开发和较简单的部署场景；更大规模的生产环境可能会使用 Kubernetes 等容器编排平台。

## 一句话总结

Docker 解决的是：

> 如何把程序连同它的运行环境，一起可靠地搬到另一台机器上。
