# 内容采集与检索平台数据库面试题库：内容规格

## 目标

生成一份可用于数据库面试复习的项目化问答文档。它以假设的内容采集与检索平台为背景，串联 MySQL InnoDB、事务、索引、缓存、日志和向量检索的取舍。

## 场景假设

- 定时采集多来源文章，经清洗、去重、分块和向量化后提供检索。
- MySQL 保存来源、文章、处理任务、处理状态和检索元数据。
- Redis 承担热点缓存、去重和任务协调。
- 对象存储保存原始正文与冷日志，Milvus 保存向量。

这些设定仅用于面试演练，不描述用户真实项目。

## 文档结构

1. 场景与数据流：建立面试回答所需的上下文。
2. 架构与表设计：解释服务职责、核心表和约束。
3. 事务、幂等与并发控制：处理重复投递、状态迁移和并发写入。
4. 索引、查询与分页：说明索引按访问路径设计，以及慢 SQL 的排查路径。
5. 缓存、读写与扩展：说明缓存一致性、读写分离与拆分的触发条件。
6. 日志、归档与恢复：区分业务数据、原始日志和检索数据的存储位置。
7. Trade-off 追问：呈现方案边界，避免绝对化结论。

## 问答格式

每题包含问题、推荐回答、追问边界。推荐回答以面试口述为目标，避免编造不可验证的数据规模、吞吐量或线上事故。

## 验收标准

- 所有问题都能对应前述场景中的一个真实工程决策。
- MySQL 机制与工程经验边界分开描述。
- 不把固定行数、固定流量或所谓大厂惯例写为通用规则。
- 与现有 MySQL 数据类型、Milvus 索引文档互补而不重复。
